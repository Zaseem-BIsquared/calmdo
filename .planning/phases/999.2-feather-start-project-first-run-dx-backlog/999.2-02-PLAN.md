# Plan 999.2-02: Auth Provider Templates and Conditional Configuration

## Objective

Create auth.ts template variants for different provider combinations selected during the creation wizard. The wizard offers: Password (always on), OTP (opt-in), GitHub OAuth (opt-in). Each combination maps to a pre-built auth.ts template that is copied into the new project. Also handle environment variable provisioning for providers that need credentials.

## Why This Matters

Auth provider selection is Question 2 of the 3-question wizard (D-02 from CONTEXT). Password is zero-config (D-08), OTP auto-enables in dev via dev mailbox (D-09), GitHub needs credentials (D-10). The template approach (Research section 5 — Approach A) is simpler and more reliable than AST modification for a 17-line file.

## must_haves

- [x] 3 auth.ts templates covering all valid provider combinations
- [x] Template selector function that picks the right template based on wizard selections
- [x] Environment variable guidance written to .env.local or displayed for Convex dashboard setup
- [x] Each template typechecks without errors
- [x] Tests verify correct template selection for all input combinations
- [x] OTP-related files (convex/otp/, convex/password/) are kept or removed based on selection

```typescript
export type AuthProvider = "password" | "otp" | "github";

export interface AuthSelection {
  providers: AuthProvider[];
  githubClientId?: string;
  githubClientSecret?: string;
}

/**
 * Determine which auth template to use based on selected providers.
 */
export function getAuthTemplate(selection: AuthSelection): string {
  // Template name -> file path mapping
  const hasOtp = selection.providers.includes("otp");
  const hasGithub = selection.providers.includes("github");

  if (hasGithub) return "password-otp-github";  // GitHub always includes OTP
  if (hasOtp) return "password-otp";
  return "password-only";
}

/**
 * Apply auth configuration to a project directory.
 * - Copies the correct auth.ts template
 * - Removes OTP/password-reset files if not needed
 * - Writes environment hints
 */
export async function applyAuthConfig(
  projectRoot: string,
  selection: AuthSelection,
): Promise<AuthConfigResult> {
  const templateName = getAuthTemplate(selection);
  const actions: string[] = [];

  // 1. Copy auth.ts template
  // Copy from scripts/create/auth-templates/{templateName}.ts -> convex/auth.ts
  actions.push(`Copied auth template: ${templateName}`);

  // 2. Remove unused provider files for password-only mode
  if (templateName === "password-only") {
    // Remove convex/otp/ directory (not needed without OTP)
    // Remove convex/password/ResendOTPPasswordReset.ts (not needed without email sending)
    // Keep convex/password/ dir if it has other files
    actions.push("Removed OTP provider files (not selected)");
    actions.push("Removed password reset files (requires OTP for email)");
  }

  // 3. Environment variable guidance
  const envHints: string[] = [];
  if (selection.providers.includes("otp")) {
    envHints.push("# Add to Convex dashboard for production email:");
    envHints.push("# RESEND_API_KEY=re_...");
    envHints.push("# In development, emails are captured by the dev mailbox at /dev/mailbox");
  }
  if (selection.providers.includes("github")) {
    if (selection.githubClientId && selection.githubClientSecret) {
      envHints.push(`AUTH_GITHUB_ID=${selection.githubClientId}`);
      envHints.push(`AUTH_GITHUB_SECRET=${selection.githubClientSecret}`);
    } else {
      envHints.push("# GitHub OAuth (add to Convex dashboard):");
      envHints.push("# AUTH_GITHUB_ID=your_client_id");
      envHints.push("# AUTH_GITHUB_SECRET=your_client_secret");
    }
  }

  // Write .env.local with hints if any env vars needed
  if (envHints.length > 0) {
    // Write to projectRoot/.env.local
    actions.push("Created .env.local with auth provider configuration hints");
  }

  return { templateName, actions, envHints };
}

export interface AuthConfigResult {
  templateName: string;
  actions: string[];
  envHints: string[];
}
```

Key decisions:
- GitHub always implies OTP (GitHub users expect full auth features)
- password-only mode removes OTP-related backend files to keep the project clean
- .env.local file written with comments for env vars that need to be set in Convex dashboard
- Actual file operations use fs module (copyFileSync, rmSync, writeFileSync)
