# Plan 999.2-04: `feather add` Command and End-to-End Integration Tests

## Objective

Implement the `feather add <feature>` CLI command that installs a feature into an existing Feather project, and the `feather remove <feature>` command for cleanup. Bundle example app files as installable packages under `templates/examples/`. Write end-to-end integration tests that verify the complete create-then-add flow.

## Why This Matters

`feather add` is the post-creation escape hatch (D-04 from CONTEXT, D-07 from CONTEXT) — developers who skip examples during the wizard or want to add features later use this command. The integration tests are the final proof that the entire Phase 999.2 system works end-to-end: create project -> add feature -> tests pass -> typecheck passes.

## must_haves

- [ ] `feather add todos` installs the todos feature with all files and wiring
- [ ] `feather add tickets` and `feather add contacts` work similarly
- [ ] `feather remove todos` cleanly removes the feature and unwires it
- [ ] Example app bundles stored in `templates/examples/{name}/`
- [ ] CLI command registered in `bin/feather.ts`
- [ ] End-to-end test: strip project -> add feature -> verify files + wiring + typecheck
- [ ] Tests verify that `feather add` + `feather remove` is a no-op (round-trip clean)

<action>
Create `bin/commands/add.ts`:

```typescript
import { Command } from "commander";
import * as fs from "node:fs";
import * as path from "node:path";
import { wireFeature } from "../../scripts/create/wiring";

export interface AddActionOptions {
  force?: boolean;       // overwrite existing files
}

/**
 * Install a feature into the current project.
 */
export function addAction(
  featureName: string,
  options: AddActionOptions,
  projectRoot: string,
): { success: boolean; message: string; filesCreated: string[] }
```

Steps:
1. Locate example bundle: `templates/examples/{featureName}/`
2. Read manifest.json
3. Check if feature already exists (if not --force, error out)
4. Copy files from bundle to project locations:
   - `frontend/` -> `src/features/{name}/`
   - `backend/` -> `convex/{name}/`
   - `schema/` -> `src/shared/schemas/`
   - `route/` -> `src/routes/_app/_auth/dashboard/`
   - `locales/` -> `public/locales/`
   - `feather.yaml` -> `src/features/{name}/feather.yaml`
5. Wire feature into project (via wireFeature from wiring.ts):
   - Add table to schema.ts
   - Add nav entry to nav.ts
   - Add i18n namespace (if applicable)
   - Add error constants (if applicable)
6. Return result with list of created files

Register in `bin/feather.ts`:
```typescript
import { addCommand } from "./commands/add";
program.addCommand(addCommand);
```

CLI usage:
```
feather add todos                # Install todos example
feather add todos --force        # Overwrite existing files
feather add todos tickets        # Install multiple
```
</action>
<acceptance_criteria>
- `bin/commands/add.ts` exists and exports `addCommand` and `addAction`
- `addAction("todos", {}, projectRoot)` copies all files from bundle to correct locations
- `addAction` calls `wireFeature` to update schema.ts, nav.ts
- `addAction` returns `{ success: true, filesCreated: [...] }` on success
- `addAction` returns `{ success: false, message: "Feature 'todos' already exists" }` when feature dir exists and --force not set
- Command registered in `bin/feather.ts` as `feather add`
- `npm run typecheck` passes
</acceptance_criteria>
</task>

- scripts/create/wiring.ts (unwireFeature utility)
- scripts/strip-to-base.ts (strip script — similar removal logic)
- templates/examples/todos/manifest.json (manifest tells us what to remove)
</read_first>
<action>
Create `bin/commands/remove.ts`:

```typescript
import { Command } from "commander";
import * as fs from "node:fs";
import * as path from "node:path";
import { unwireFeature } from "../../scripts/create/wiring";

/**
 * Remove a feature from the current project.
 */
export function removeAction(
  featureName: string,
  options: { confirm?: boolean },
  projectRoot: string,
): { success: boolean; message: string; filesRemoved: string[] }
```

Steps:
1. Check feature exists in project
2. If not --confirm, display warning: "This will remove all files for '{featureName}'. Are you sure?"
3. Remove files (reverse of add):
   - Delete `src/features/{name}/`
   - Delete `convex/{name}/`
   - Delete `src/shared/schemas/{name}.ts`
   - Delete route file
   - Delete locale files
4. Unwire feature (via unwireFeature):
   - Remove table from schema.ts
   - Remove nav entry from nav.ts
5. Return result with list of removed files

Register in `bin/feather.ts`:
```typescript
import { removeCommand } from "./commands/remove";
program.addCommand(removeCommand);
```

CLI usage:
```
feather remove todos             # Remove with confirmation prompt
feather remove todos --confirm   # Skip confirmation (for scripts)
```
</action>
<acceptance_criteria>

- `removeAction("todos", { confirm: true }, projectRoot)` deletes all feature files

- `removeAction` returns `{ success: true, filesRemoved: [...] }` on success
- `removeAction` returns `{ success: false, message: "Feature 'todos' not found" }` when feature doesn't exist
