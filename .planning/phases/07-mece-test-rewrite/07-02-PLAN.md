# Plan 07-02: Utils, Shared, and UI Test Cleanup + Coverage Config

## Objective

Clean up utility/shared/UI test files for philosophy compliance, delete stub test files, update coverage config, and centralize nav tests. These tests are already largely compliant — this is a light-touch pass.

## Why This Matters

Utils/shared/UI tests form the base layer — pure functions and hooks that don't need integration infrastructure. Most are already well-structured, but need naming standardization, test matrix comments, and nav test centralization to eliminate duplication with feature tests.

## must_haves
- [ ] All utils/shared/UI test files have MECE test matrix comments
- [ ] Test names follow philosophy convention
- [ ] `nav.test.ts` is the single source of truth for nav item tests (no duplicates in feature tests)
- [ ] Stub test files deleted (subtasks, work-logs frontend)
- [ ] `vitest.config.ts` coverage config cleaned up (remove `errors.ts` from `include` since it's in `exclude`)
- [ ] `npm test` passes with 100% coverage
