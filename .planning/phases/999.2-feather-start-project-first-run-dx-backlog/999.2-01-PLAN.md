# Plan 999.2-01: Base Branch Strip Script and CI Workflow

## Objective

Build a Node.js script that programmatically strips all feature code from the main branch to produce a clean `base` branch containing only infrastructure (auth, dashboard, onboarding, settings, uploads, UI components, generators, build config). A GitHub Actions workflow runs this script on every push to main, keeping the base branch always in sync.

## Why This Matters

The hybrid approach (D-12 from CONTEXT) requires a minimal base branch that developers clone via `npm create feather`, then scaffold selected features with `feather add`. Without automated stripping, the base branch drifts from main and breaks. CI automation makes this maintenance-free.

## must_haves

- [x] Strip script removes all feature directories (frontend + backend) for: tasks, projects, subtasks, work-logs, activity-logs, todos, tickets, contacts
- [x] Strip script cleans wiring files: schema.ts (remove feature tables), nav.ts (remove feature entries), i18n.ts (remove feature namespaces), errors.ts (remove feature groups)
- [x] Strip script removes feature routes from `src/routes/_app/_auth/dashboard/`
- [x] Strip script removes feature schemas from `src/shared/schemas/` (keeps username.ts)
- [x] Strip script removes feature locale files from `public/locales/`
- [x] Strip script is idempotent — running twice produces identical output
- [x] GitHub Actions workflow auto-runs on push to main and force-pushes to base branch
- [x] Tests verify correct stripping of each wiring file
- [x] `npm run typecheck` passes on the stripped output

<action>
Create `scripts/strip-to-base.ts`:

**Part 1: Directory and file deletion**
```typescript
function stripFeatureDirectories(projectRoot: string, config: StripConfig): string[]
```
For each feature in config.features:
1. Delete `src/features/{featureName}/` directory (recursively)
2. Delete `convex/{backendName}/` directory (handle naming: work-logs -> workLogs backend dir, activity-logs -> activityLogs backend dir)
3. Delete feature schema file from `src/shared/schemas/{name}.ts`
4. Delete feature route files from `src/routes/_app/_auth/dashboard/`
5. Delete feature locale files from `public/locales/{lang}/{name}.json` for all languages
6. Delete feature feather.yaml files
7. Return list of deleted paths

**Part 2: Schema.ts surgical editing**
```typescript
function stripSchemaFile(schemaPath: string, config: StripConfig): void
```
Read `convex/schema.ts` and:
1. Remove import lines that reference stripped feature schemas (imports from `../src/shared/schemas/tasks`, etc.)
2. Remove defineTable blocks for stripped features (tasks, projects, subtasks, workLogs, activityLogs, todos, tickets, contacts)
3. Keep: authTables spread, users table, devEmails table
4. Write cleaned file back

Strategy: Line-by-line processing. For imports, remove lines containing paths to stripped schema files. For table definitions, track brace depth to remove entire `tableName: defineTable({...}).index(...)` blocks including chained index calls.

**Part 3: Nav.ts surgical editing**
```typescript
function stripNavFile(navPath: string, config: StripConfig): void
```
Read `src/shared/nav.ts` and remove nav item objects whose `to` path matches stripped feature routes:
- Remove: `/dashboard/tasks`, `/dashboard/team-pool`, `/dashboard/projects`, `/dashboard/todos`, `/dashboard/tickets`, `/dashboard/contacts`
- Keep: `/dashboard`, `/dashboard/settings`, `/dev/mailbox`

Strategy: Parse the navItems array, filter out entries with stripped routes, write back.

**Part 4: Errors.ts editing**
```typescript
function stripErrorsFile(errorsPath: string, config: StripConfig): void
```
Remove error constant groups for stripped features. Keep infrastructure error groups.

**Part 5: Main orchestrator**
```typescript
export async function stripToBase(projectRoot: string, config?: StripConfig): Promise<StripResult>
```
1. Load config (default or provided)
2. Run directory/file deletion
3. Run schema editing
4. Run nav editing
5. Run errors editing
6. Return summary: { deleted: string[], modified: string[], errors: string[] }

Make the script executable: `if (import.meta.url === ...) { stripToBase(process.cwd()) }`
</action>

<action>
Create `.github/workflows/update-base-branch.yml`:

```yaml
name: Update Base Branch
on:
  push:
    branches: [main]

jobs:
  update-base:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # full history for branch operations

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci

      - name: Strip feature code
        run: npx tsx scripts/strip-to-base.ts

      - name: Verify stripped project
        run: |
          npm run typecheck
          npm test -- --silent 2>/dev/null || echo "Tests may fail on base (expected — feature tests removed)"

      - name: Commit and push to base branch
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git checkout -B base
          git add -A
          git commit -m "chore: auto-generate base branch from main ($(git rev-parse --short HEAD~1))" || true
          git push origin base --force
```

Key design decisions:
- Force-push to base — base branch is always regenerated, not incrementally updated
- Full checkout (fetch-depth: 0) so we can create the base branch
- npm ci for clean install
- Typecheck after stripping to catch broken imports
- Test run is best-effort (feature tests will be deleted, so some tests may fail — this is expected)
- Bot commit so it's clear this is automated
</action>
