## Decisions Made
- smartAdd uses `plop.renderString` to compile templates (inherits all registered Handlebars helpers)
- Wiring actions use direct string manipulation with pattern matching for precise insertion points (more reliable than Plop's built-in modify for multi-point insertions)
- appendToNav inserts before Settings entry for consistent ordering
- All wiring actions are idempotent -- they check if the feature is already wired before modifying
- Schema wiring adds zodToConvex import only when not already present
