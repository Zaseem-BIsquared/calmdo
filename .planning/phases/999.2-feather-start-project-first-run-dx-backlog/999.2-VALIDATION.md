# Phase 999.2 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | vitest (unit/integration), manual E2E (create script in temp dir) |
| **Config file** | `vitest.config.ts` |
| **Quick run command** | `npm test -- --silent` |
| **Full suite command** | `npm test` |
| **Estimated runtime** | ~30 seconds |

---

## Sampling Rate

- **After every task commit:** Run `npm test -- --silent`
- **After wiring changes (schema, nav, i18n):** Run `npm run typecheck`
- **After create script changes:** Run create script in temp dir, verify output
- **After auth template changes:** Typecheck each variant

---

## Testable Claims from Research

| # | Claim | How to Verify | Required Evidence |
|---|-------|--------------|-------------------|
| V1 | Create script produces working project | Run script in temp dir, `npm test` in output | Exit code 0, all tests pass |
| V2 | Base branch has no feature code | Grep for feature dirs, check schema tables | No hits for feature paths, only infra tables in schema |
| V3 | `feather add` installs feature correctly | Run add command, check files + wiring | Feature files exist, schema updated, nav updated, tests pass |
| V4 | Auth templates produce valid auth.ts | `tsc --noEmit` on each variant | No type errors |
| V5 | CI strip job is idempotent | Run strip twice, diff | Empty diff |
| V6 | Post-creation verification catches issues | Break something, run verify | Script reports failure |
