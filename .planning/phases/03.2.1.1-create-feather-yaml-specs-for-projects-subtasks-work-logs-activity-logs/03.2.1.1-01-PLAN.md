# Plan 03.2.1.1-01: CalmDo feather.yaml Specs

## Objective

Upgrade all 5 CalmDo entity feather.yaml specs to complete, aspirational contracts that describe the desired end state of each feature. Reconcile tasks feather.yaml with tasks.gen.yaml to ensure no information is lost. Each spec follows the gold standard structure from tasks/feather.yaml (the reference) with all applicable sections: identity, fields, timestamps, behaviors, access, views, filteredViews, relationships, detailView, derivedData, hooks, indexes. Sections the generator does not yet consume are marked with `# TODO: generator pending`.

## Why This Matters

These YAML specs become the single source of truth for what each CalmDo feature should do when fully realized. They serve as test fixtures for Phase 999.4 (generator upgrade to consume feather.yaml) — defining what the generator should produce. Gaps between spec and current code are intentional and represent the generator's backlog.

## must_haves

- [ ] tasks/feather.yaml reconciled with tasks.gen.yaml — no information lost, gen.yaml differences merged
- [ ] projects/feather.yaml complete with all sections (currently missing description field, identity, hooks)
- [ ] subtasks/feather.yaml complete with promotion behavior, detailView, hooks
- [ ] work-logs/feather.yaml complete with auditTrail behavior, proper camelCase name
- [ ] activity-logs/feather.yaml complete with kind:system flag, actor upgraded to belongs_to relationship
- [ ] All feather.yaml fields match convex/schema.ts ground truth
- [ ] All feather.yaml indexes match convex/schema.ts indexes
- [ ] Aspirational sections (detailView, derivedData, hooks) marked with `# TODO: generator pending`
