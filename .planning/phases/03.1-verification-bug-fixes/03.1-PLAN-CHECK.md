### WARNING 1: Plan 02 Task 1 verify command may under-test the changes

The verify command runs `npx vitest run src/features/auth/ --reporter=verbose -x`. This covers `PasswordForm.test.tsx` but the action also changes `_layout.index.tsx` (login page), `_layout.tsx` (login layout), and `PasswordResetForm.tsx`. None of these have their tests targeted in the verify command. The login page changes (provider hiding, email lift, layout centering) have no automated test coverage planned.

The `autonomous: false` flag on Plan 02 signals the executor knows human review is needed for the visual pieces — the checkpoint task (Task 3) catches this. But the automated verify for Task 1 will pass even if the layout changes introduce a runtime crash, because it only runs the auth component tests.

```yaml
issue:
  plan: "03.1-02"
  dimension: "task_completeness"
  severity: warning
  description: "Task 1 verify only runs src/features/auth/ tests but changes three other files: _layout.index.tsx, _layout.tsx, PasswordResetForm.tsx. A type error in those files would not surface until Task 3 human verify."
  task: 1
  fix_hint: "Add tsc --noEmit to the automated verify command: && npx tsc --noEmit 2>&1 | tail -20. This catches prop type errors across all modified files without requiring a full test suite."
```

### WARNING 2: `_app/dev/` directory creation may need a parent route file depending on TanStack Router config

The plan moves `mailbox.tsx` from `_app/_auth/dev/` to `_app/dev/`. The existing `_auth/dev/` directory has no parent layout file — just `mailbox.tsx` directly. The new `_app/dev/` directory will be created from scratch with the same pattern.

TanStack Router v1 file-based routing: a flat file at `_app/dev/mailbox.tsx` generates a route `/_app/dev/mailbox`. Since `_app.tsx` exists as the parent layout, this should resolve correctly. No explicit parent layout for `dev/` is needed.

However, the plan says `src/routes/_app/dev/mailbox.tsx` will be the new file path but does not mention creating the `dev/` directory itself (it does not exist yet at `src/routes/_app/dev/`). The executor should create the directory. This is implicit from the file path but worth calling out.

```yaml
issue:
  plan: "03.1-02"
  dimension: "task_completeness"
  severity: warning
  description: "src/routes/_app/dev/ directory does not exist yet. Task 2 action says to create the file but does not mention mkdir. Executor should handle this, but the action could be explicit."
  task: 2  # (Plan 01, Task 2)
  plan: "03.1-01"
  fix_hint: "Add step: 'Create src/routes/_app/dev/ directory if it does not exist' to Task 2 action"
```

### WARNING 3: Issue 5 (dev mailbox captures no emails on local) is acknowledged but not fixed

Plan 01 Task 2 notes that issue 5 is not a code problem — the root cause is whether `CONVEX_URL` is set in the environment when the OTP handler runs. This is a valid call. With the Resend-skip fix in Task 1, the OTP/reset flows complete, and `storeDevEmail` should run because `process.env.DEV_MAILBOX !== "false"` by default.

But the NOTE also says "if emails still don't appear, the root cause is likely that `DEV_MAILBOX` env var needs to be set." That sentence is slightly wrong — the default behavior already stores when the flag is absent. The actual uncertainty is whether `CONVEX_URL` is available in the Convex runtime action context. This is important because `storeDevEmail` silently returns if `CONVEX_URL` is missing.

This is not a plan failure — the assessment is reasonable given the information available. But the executor should watch for this during the checkpoint in Plan 02 Task 3 (step: "OTP and password reset without Resend key should capture to dev mailbox").

```yaml
issue:
  plan: "03.1-01"
  dimension: "verification_derivation"
  severity: warning
  description: "Issue 5 (dev mailbox captures no emails) is deferred with a NOTE rather than explicitly closed. Success criterion 1 requires dev mailbox captures emails. If CONVEX_URL is not available in the Convex action environment, storeDevEmail silently no-ops and the criterion will fail."
  task: 2
  fix_hint: "In Task 2 done criteria, add: 'Confirm CONVEX_URL env var is available in ResendOTP.ts context (check convex.json or .env.local). If not, storeDevEmail will silently skip.'"
```
