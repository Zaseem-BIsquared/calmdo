# Phase 999.4 Plan 2: Cross-Entity Panel Templates + Detail Page Update Summary

**One-liner:** Four Handlebars panel templates (checklist/timeline/table/cards) with conditional readonly/inline rendering, plus pipeline integration via `renderCrossEntityPanels()` — CalmDo tasks page now generates real panel UI from YAML.

## What Was Built

### Panel Templates (4 new .hbs files)

**`checklist-panel.tsx.hbs`** — Inline checkable list with:
- `useQuery` / `useMutation` imports conditional on `readonly` flag
- Checkbox toggle button, drag handle (GripVertical), inline add form
- Add button only rendered when `inline=true` and `readonly=false`
- `Plus`/`Trash2`/`useMutation` excluded from output when readonly

**`timeline-panel.tsx.hbs`** — Chronological entries with:
- Vertical timeline (`border-l-2 border-gray-200`) + timestamp dots
- `formatRelativeTime()` helper for relative timestamps
- Optional duration field for work-logs, event type badge for activity-logs
- Inline textarea add form at top when `inline=true`

**`table-panel.tsx.hbs`** — Sortable data table with:
- Columns from `{{columns}}` array with `toTitleCase()` headers
- Sort state with ChevronUp/Down indicators
- Action column (Pencil, Trash2) when not readonly
- Pagination when > 20 rows

**`cards-panel.tsx.hbs`** — Responsive grid with:
- `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` layout
- Dynamic field rendering from first 4 non-internal fields
- Status badge if entity has status field

### Panel Renderer (`panel-renderer.tsx.hbs`)

Aggregator component that uses `{{#each relatedRecords}}` to generate imports and JSX for all panels:
- `ifEq display "checklist/timeline/table/cards"` selects correct import
- Exported as `{{parentPascal}}Panels` component
- Accepts `parentId: Id<"{{parentEntity}}">` prop

### Pipeline Integration

**`render.ts`** — New `renderCrossEntityPanels()` function:
- Imports `resolveDetailPage` from cross-entity resolver
- Iterates `relatedRecords`, renders matching panel template per display type
- Renders panel-renderer template with full `DetailPageContext`
- Returns Map of `components/${source}-${display}-panel.tsx` → content

**`scaffold.ts`** — Updated `scaffoldFeature()`:
- Detects `config.detailView` presence
- Calls `renderCrossEntityPanels()` with `options.relatedYamls ?? new Map()`
- Merges panel outputs into the main rendered Map
- `ScaffoldOptions.relatedYamls?: Map<string, FeatureYaml>` — optional, backward compatible

**`detail-page.tsx.hbs`** — Updated:
- Conditional import: `{{#if detailView.relatedRecords}}import { {{pascalCase name}}Panels }...{{/if}}`
- `panelsSection` variable rendered after `detailContent`
- Panels rendered below field groups in page display mode
