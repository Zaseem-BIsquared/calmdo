# Phase 999.4 Plan 01: Pipeline Template Integration Summary

Replaced 5 inline stub generators in `scaffold.ts` with real Handlebars template rendering using all 26 templates in `templates/feature/`. The pipeline now produces the same quality output as the Plop generators, and `npx feather generate` works as a CLI command.

## What Was Built

**render.ts** — standalone Handlebars template renderer:
- `registerStandaloneHelpers(hbs)` — registers 35+ helpers adapted from `handlebars-helpers.js` plus `change-case` case-transformation helpers
- `renderTemplate(path, data)` — reads and compiles a single .hbs template
- `renderFeatureTemplates(config, templateDir)` — builds full template data context and renders all applicable templates, returning `Map<outputKey, renderedContent>`

**scaffold.ts rewrite** — replaced 5 stubs with `renderFeatureTemplates()` call. Maps 25 output keys to correct absolute paths (backend, frontend, locales). Prepends `@generated DO NOT EDIT` header in generated mode.

**package.json** — added `"bin": { "feather": "./bin/feather.ts" }` and `tsx` to devDependencies.

**Tests** — 39 new tests:
- render.test.ts: 30 tests (helper registration, individual helper behavior, template rendering, full feature rendering with fixture)
- pipeline.test.ts: expanded from 5 to 13 tests (real content verification, mode switching, wiring)
- generate.test.ts: 1 new test (dry-run >= 20 files)
