# Plan 999.2-03: Create-Feather Wizard — Project Scaffolding with Interactive Prompts

## Objective

Build the `create-feather` entry point that developers run via `npm create feather my-project` (or locally via `npx tsx scripts/create/index.ts my-project`). The wizard asks 3 questions (project name, auth providers, example apps), clones the base branch, applies branding, configures auth, and scaffolds selected features. The result is a working project ready for `npm run dev`.

## Why This Matters

This is the primary user-facing deliverable of Phase 999.2 — the "one command to a working project" experience. Decisions D-01 through D-04 from CONTEXT define the wizard shape. The Tableau Standard (D-13 from 999.1) requires smart defaults: Password auth by default, all examples installed by default, app name derived from project directory name.

## must_haves

- [ ] 3-question interactive wizard: project name, auth providers (multi-select), example apps (multi-select)
- [ ] Clone base branch into target directory
- [ ] Apply branding (app name, package name) using setup.ts patterns
- [ ] Configure auth via template selection (Plan 02)
- [ ] Scaffold selected example apps via generator pipeline
- [ ] Run `npm install` and `npx convex dev --once` for verification
- [ ] Squash-reset for clean git history (one initial commit)
- [ ] Success message with clear next steps
- [ ] Tests cover prompts logic and scaffold operations

```typescript
  projectName: string;
  authProviders: ("password" | "otp" | "github")[];
  exampleApps: string[];
}

export interface PromptOptions {
  defaultName?: string;        // from CLI arg
  nonInteractive?: boolean;    // for testing — use all defaults
}

/**
 * Available example apps with complexity labels for the wizard display.
 */
export const EXAMPLE_APPS = [
  { name: "todos", label: "Todos", complexity: "Simple CRUD" },
  { name: "tickets", label: "Tickets", complexity: "Status transitions + priority" },
  { name: "contacts", label: "Contacts", complexity: "Multi-field entity" },
] as const;

/**
 * Available auth providers for the wizard.
 */
export const AUTH_PROVIDERS = [
  { id: "password" as const, label: "Password", note: "Zero-config, always enabled", default: true },
  { id: "otp" as const, label: "OTP (Email codes)", note: "Auto-works in dev via dev mailbox", default: false },
  { id: "github" as const, label: "GitHub OAuth", note: "Needs OAuth app credentials", default: false },
] as const;

/**
 * Run the 3-question creation wizard.
 */
export async function runWizard(options: PromptOptions = {}): Promise<WizardAnswers>
```

**Question 1: Project name** (required, no default)
```
? Project name: my-project
```
If provided via CLI arg (options.defaultName), skip the prompt.

**Question 2: Auth providers** (multi-select, default: Password only)
```
? Auth providers (space to toggle, enter to confirm):
  [x] Password (zero-config, always enabled)
  [ ] OTP — Email codes (auto-works in dev via dev mailbox)
  [ ] GitHub OAuth (needs OAuth app credentials)
```
Implementation note: readline doesn't natively support multi-select. Use a numbered list approach:
```
Auth providers (Password is always included):
  1. [default] Password only
  2. Password + OTP (email codes — auto-works in dev)
  3. Password + OTP + GitHub OAuth
Enter choice (1-3) [1]:
```
If GitHub selected, follow up with:
```
GitHub OAuth Client ID (or press Enter to skip):
GitHub OAuth Client Secret (or press Enter to skip):
```

**Question 3: Example apps** (multi-select, default: all)
```
Example apps to install (all included by default):
  1. [default] All examples (todos, tickets, contacts)
  2. None — start with blank project
  3. Custom — choose which to include
Enter choice (1-3) [1]:
```
If "Custom" selected:
```
Include todos? (Y/n):
Include tickets? (Y/n):
Include contacts? (Y/n):
```

Smart defaults (Tableau Standard):
- Project name: required, no inference (D-02: "required, no default")
- App display name: derived from project name (my-project -> My Project) per D-03
- Auth: Password only (zero friction)
- Examples: All installed (D-05: "Default: install ALL existing example apps")

For `nonInteractive` mode (testing), return all defaults with the provided name.

- The scaffold function copies the spec, then invokes the Plop generator programmatically

**Pre-built approach (recommended for v1):**

This is the inverse of the strip script from Plan 01 — where strip removes features from wiring files, scaffold adds them.

Create `scripts/create/index.ts` — the main entry point:

```typescript
#!/usr/bin/env npx tsx

import { runWizard } from "./prompts";
import { cloneBaseProject } from "./clone";
import { applyBranding } from "./branding";
import { applyAuthConfig } from "./auth-config";
import { scaffoldExamples } from "./scaffold";

const REPO_URL = "https://github.com/siraj-samsudeen/feather-starter-convex.git";

async function main(): Promise<void> {
  // Parse CLI args
  const projectName = process.argv[2];

  console.log("");
  console.log("  Feather — Create New Project");
  console.log("  ─────────────────────────────");
  console.log("");

  // Step 1: Run wizard
  const answers = await runWizard({ defaultName: projectName });

  // Step 2: Clone base branch
  console.log(`\n  Cloning Feather base into ${answers.projectName}...`);
  await cloneBaseProject({
    repoUrl: REPO_URL,
    branch: "base",
    targetDir: answers.projectName,
    projectName: answers.projectName,
  });

  const projectRoot = path.resolve(answers.projectName);

  // Step 3: Apply branding
  console.log("  Applying branding...");
  const appDisplayName = toTitleCase(answers.projectName);
  const filesUpdated = applyBranding({
    projectRoot,
    projectName: answers.projectName,
    appDisplayName,
  });
  console.log(`    Updated ${filesUpdated} files`);

  // Step 4: Configure auth
  console.log("  Configuring authentication...");
  const authResult = await applyAuthConfig(projectRoot, {
    providers: answers.authProviders,
    githubClientId: answers.githubClientId,
    githubClientSecret: answers.githubClientSecret,
  });
  console.log(`    Template: ${authResult.templateName}`);

  // Step 5: Scaffold example apps
  if (answers.exampleApps.length > 0) {
    console.log(`  Installing example apps: ${answers.exampleApps.join(", ")}...`);
    const scaffoldResult = await scaffoldExamples({
      projectRoot,
      exampleApps: answers.exampleApps,
    });
    console.log(`    Installed ${scaffoldResult.installed.length} apps`);
  }

  // Step 6: Install dependencies
  console.log("  Installing dependencies...");
  execSync("npm install", { cwd: projectRoot, stdio: "inherit" });

  // Step 7: Initialize Convex
  console.log("  Initializing Convex...");
  try {
    execSync("npx convex dev --once", { cwd: projectRoot, stdio: "inherit" });
  } catch {
    console.log("    Convex init completed (errors above may be expected on first run).");
  }

  // Step 8: Restore convex/tsconfig.json (Convex CLI overwrites it)
  try {
    const tsconfigPath = path.join(projectRoot, "convex/tsconfig.json");
    // Copy from template or restore from git
    execSync("git restore convex/tsconfig.json", { cwd: projectRoot });
  } catch {
    // Not critical
  }

  // Step 9: Squash to clean history
  execSync("git add -A", { cwd: projectRoot });
  execSync(`git commit -m "Initial commit — ${appDisplayName}"`, { cwd: projectRoot });

  // Step 10: Success message
  console.log("");
  console.log("  ────────────────────────────────────────");
  console.log(`  ${appDisplayName} is ready!`);
  console.log("  ────────────────────────────────────────");
  console.log("");
  console.log(`  cd ${answers.projectName}`);
  console.log("  npm run dev");
  console.log("");
  if (authResult.envHints.length > 0) {
    console.log("  Auth notes:");
    for (const hint of authResult.envHints) {
      console.log(`    ${hint}`);
    }
    console.log("");
  }
  console.log("  Add more features later: npx feather add <feature>");
  console.log("");
}

main().catch((err) => {
  console.error("\n  Create failed:", err.message);
  process.exit(1);
});
```
